## Overview

2md exposes a set of standard discovery endpoints so that AI agents, crawlers, and tool registries can automatically understand the service's capabilities and retrieve generated Skill Bundles without human configuration.

The implementation follows the [llmstxt.org](https://llmstxt.org) specification for `llms.txt` files and the OpenAI plugin manifest convention for `/.well-known/ai-plugin.json`.

## Endpoints

### GET /llms.txt

The standard entry point for AI agents. Returns a concise machine-readable description of the service and its capabilities, following the llmstxt.org specification.

```
GET /llms.txt
```

**Response:** `text/plain; charset=utf-8`

The file lists the service name, a brief description, and links to key documentation pages. All `{{BASE_URL}}` placeholders in the stored file are replaced at runtime with the actual server host, so URLs are always correct regardless of deployment environment.

---

### GET /llms-full.txt

The verbose entry point for AI agents that need complete context — includes full API documentation, all endpoint descriptions, parameter tables, and example requests.

```
GET /llms-full.txt
```

**Response:** `text/plain; charset=utf-8`

Like `/llms.txt`, all `{{BASE_URL}}` placeholders are replaced with the runtime host at serve time.

---

### GET /skills/:skillName.md

Retrieve a generated AI Skill Bundle by the site hostname slug. Skill Bundles are created by running `POST /api/agentify` against a documentation site.

```
GET /skills/html2md-scraper.md
GET /skills/:skillName.md
```

**Path Parameters:**

| Parameter | Description |
|---|---|
| `skillName` | The skill name slug. Only alphanumeric characters, hyphens, and underscores are allowed. |

**Response (200):** `text/markdown; charset=utf-8` — the `SKILL.md` file for the named skill.

**Error (400):**
```
Invalid skill name
```

**Error (404):**
```
Skill not found
```

Skill files are stored at `.agents/skills/:skillName/SKILL.md` on the server after a successful Agentify run.

## Dynamic BASE_URL Replacement

The `llms.txt` and `llms-full.txt` files are stored on disk with `{{BASE_URL}}` placeholders. At request time, the server replaces every occurrence with the actual runtime base URL derived from the incoming request's `x-forwarded-proto` and `host` headers:

```
https://{{BASE_URL}}/api/convert
↓ at runtime
https://2md.traylinx.com/api/convert
```

This means the same server binary can be deployed to any domain and the discovery files will always contain correct, absolute URLs without any rebuild or manual configuration step.

## llmstxt.org Specification

The `/llms.txt` endpoint implements the [llmstxt.org](https://llmstxt.org) community specification for making web services legible to Large Language Models. The spec defines:

- A plain-text file at `/llms.txt` with a title, optional description, and a list of URLs
- A verbose companion at `/llms-full.txt` for agents that need complete context
- Human-readable Markdown formatting so the file is useful to both people and models

<Info>
  The full specification is available at [llmstxt.org](https://llmstxt.org). 2md implements both the standard `/llms.txt` and the `/llms-full.txt` companion as described in the spec.
</Info>



```bash Fetch agents manifest
curl https://2md.traylinx.com/agents.json
```

