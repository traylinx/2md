The **Crawl** tool is engineered to construct large-scale knowledge bases from public websites. Unlike a single-page conversion, Crawl separates discovery from execution, allowing you to explicitly curate the token budget of your LLM corpus before downloading it.

## Accessing Crawl

Navigate to **[https://2md.traylinx.com/crawl](https://2md.traylinx.com/crawl)** via the secondary tab on the main dashboard.

## Multi-Page Ingestion Workflow

<Steps>
  <Step title="Map the Domain">
    Paste the root URL of target documentation (e.g., `https://docs.stripe.com/api`). Set your discovery bounds:

    | Parameter | Default | Purpose |
    |-----------|---------|-------------|
    | `Depth` | `3` | How many recursive link layers to trace from the origin. |
    | `Max Pages` | `50` | Failsafe limit to prevent runaway crawling on massive sites. |
  </Step>
  <Step title="Run Discovery">
    Click **Discover**. The proxy spiders the target domain executing a `treeOnly` algorithm. It reconstructs the topological map of the site visually so you can perceive exactly how the knowledge base is structured.
  </Step>
  <Step title="Curate Your Corpus">
    The dashboard presents a checklist of every discovered URL. Deselect irrelevant pages (Pricing, Author bios, Legacy API versions) to protect your vector database from poisoning and save execution tokens.
  </Step>
  <Step title="Execute Batch Conversion">
    Provide an email address for asynchronous delivery. Click **Convert Selected**.
    
    The engine aggressively spins up massive parallel browser clusters to hit the selected endpoints concurrently.
  </Step>
  <Step title="Download Zip Archive">
    The service completes the batch processing securely on the backend. You receive a cryptographic link to download a `.zip` archive containing fully sanitized Markdown shards ready for ingestion into Pinecone, Qdrant, or basic local storage.
  </Step>
</Steps>

## Why separate discovery from conversion?

Blindly crawling an entire domain severely dilutes vector search performance and wastes your model's context window with navigational noise. The 2-step methodology ensures you only ingest the exact technical boundaries you desire.

## SaaS Implementation Examples

### 1. Internal Agent Training
You want your internal customer service AI to know the exact shipping procedures detailed on a third-party logistics provider's site.
- **Workflow**: Point Crawl at the vendor's FAQ subdomain. Discover 30 pages. Deselect their marketing blocks. Extract to an archive and push directly to your internal OpenAI Assistant's file search storage.

### 2. Large Scale Competitor Analysis
You need to pass a competitor's entire feature catalog to Claude for benchmarking.
- **Workflow**: Crawl their `/features` endpoint with depth `2`. Wait 3 minutes for the asynchronous cluster execution, download the zip archive, and drag-and-drop the entire unzipped directory directly into the Claude console.

## Asynchronous Architecture
For enterprise crawls exceeding 15 target endpoints, the SaaS dashboard automatically routes your request directly into the backend background job queue. This guarantees your session won't timeout, allowing you to close your laptop and receive the archive asynchronously via email.
