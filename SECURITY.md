# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in html2md, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **security@traylinx.com**

Include:
- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (optional)

## Response Timeline

- **Acknowledgment**: within 48 hours
- **Assessment**: within 7 days
- **Fix & Disclosure**: coordinated with reporter

## Scope

This policy covers the html2md open-source codebase. For vulnerabilities in the hosted service at `2md.traylinx.com`, please also email the address above.

|---------|-----------|
| Latest  | ✅        |

Thank you for helping keep html2md secure.
